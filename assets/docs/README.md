# MyFestival (ระบบฝากคำอวยพรตามเทศกาล)

MyFestival คือเว็บแอปพลิเคชันสำหรับฝากส่งคำอวยพรและคำอธิษฐานดีๆ ตามช่วงเทศกาลประเพณีต่างๆ ของไทย โดยผู้เขียน (Contributor) สามารถสร้างสรรค์ข้อความอวยพรร่วมกะเกณฑ์ตรวจสอบผ่านแอดมิน เพื่อนำไปเพิ่มเข้าสู่คลังสำหรับสุ่มแจกจ่ายความสุขให้ผู้เข้าชมทั่วไป (Guest) และสมาชิกทั่วไป (Member) ได้จับสุ่มอ่าน

---

## 🎨 สไตล์การออกแบบ (Visual Style)
แอปพลิเคชันได้รับการจัดทำขึ้นภายใต้ธีม **Sketchbook Style** (การ์ตูนสีไม้กระดาษเขียนมือ) ที่ให้ความรู้สึกอบอุ่น สบายตา และเป็นกันเอง หลีกเลี่ยงรูปแบบ Cyberpunk, Glassmorphism และ Neumorphism โดยสิ้นเชิง

## ⚙️ การตั้งค่าฐานข้อมูล Supabase (Supabase Setup)

คุณจำเป็นต้องมีบัญชีผู้ใช้งาน Supabase เพื่อรันระบบเซิร์ฟเวอร์ฐานข้อมูลหลังบ้าน โดยมีขั้นตอนการดำเนินงานดังนี้:

### 1. การทำ Database Migration
นำคำสั่งในโฟลเดอร์ `database/` ไปรันในหน้า **SQL Editor** ของ Supabase Console ตามลำดับดังต่อไปนี้:
1. รันไฟล์ [schema.sql](file:///d:/make_website/MyFestival/database/schema.sql) เพื่อสร้างตารางฐานข้อมูลและทริกเกอร์สร้างโปรไฟล์อัตโนมัติ
2. รันไฟล์ [rls.sql](file:///d:/make_website/MyFestival/database/rls.sql) เพื่อตั้งค่าความปลอดภัยระดับแถวข้อมูล (Row Level Security) และทริกเกอร์ส่งเรื่องพิจารณารายงานให้แอดมิน
3. รันไฟล์ [seed.sql](file:///d:/make_website/MyFestival/database/seed.sql) เพื่อใส่ข้อมูลเริ่มต้นเทศกาล 8 เทศกาลประเพณี และคำอวยพรรุ่นแรก พร้อมสร้างไอดี Contributor ต้นแบบสำหรับใช้งานระบบ

### 2. ล็อกระบบความปลอดภัย Admin ล็อก
- บัญชี Admin ของระบบจะจำกัดไว้ที่ผู้ใช้อีเมล `6nathan.dev@gmail.com` เท่านั้น
- เมื่อทำการสมัครสมาชิกใหม่ หรือล็อกอินผ่าน Google ด้วยอีเมลดังกล่าว ระบบจะปรับบทบาทเป็น `admin` ในตาราง `profiles` อัตโนมัติด้วยคำสั่ง trigger ด้านความปลอดภัยระดับเซิร์ฟเวอร์
- RLS Policy ป้องกันไม่ให้สมาชิกคนอื่นปรับยกระดับบทบาทตัวเองเป็น Admin หรือ Contributor ได้เองจากหน้าบ้าน

## 💻 การติดตั้งและเปิดรันโปรเจกต์ภายในเครื่อง (Local Development)

### 1. ติดตั้ง Dependencies
ใช้ Node.js และ NPM ในการรันเซิร์ฟเวอร์จำลองการทำงาน:
```bash
# เดินทางเข้าสู่โฟลเดอร์โปรเจกต์
cd MyFestival

# ติดตั้งแพ็กเกจ (Vite)
npm install
```

### 2. กำหนดตัวแปรสภาพแวดล้อม (Environment Variables)
สร้างไฟล์ `.env` ไว้ที่โฟลเดอร์รูทของโครงการ หรือกรอกผ่านแผงหน้ากากการตั้งค่า UI ในหน้าเว็บเมื่อเข้าใช้งานครั้งแรกได้ทันที:
```env
VITE_SUPABASE_URL=https://ของคุณ.supabase.co
VITE_SUPABASE_ANON_KEY=รหัสคีย์anonสาธารณะของคุณ
```

### 3. เปิดรัน Development Server
รันคำสั่งจำลองเครื่องเซิร์ฟเวอร์สำหรับเขียนและดูผลงานแบบเรียลไทม์:
```bash
npm run dev
```
เปิดบราวเซอร์ไปที่ลิงก์ที่แสดงใน Terminal (เช่น `http://localhost:5173`)

## 🚀 การ Deploy ขึ้นสู่โปรดักชัน Vercel (Deploy Vercel)

โปรเจกต์นี้ได้รับการตั้งค่าไฟล์ [vercel.json](file:///d:/make_website/MyFestival/vercel.json) เพื่อให้รองรับระบบ SPA Routing อย่างถูกต้อง (ส่งมอบหน้าคำแชร์ประเภท `/message/abc123` กลับไปยัง `index.html` เพื่อให้ Router.js แยกพารามิเตอร์ประมวลผล):

1. สมัครใช้งานบน Vercel และเชื่อมต่อกับคลัง Git Repository
2. กดสร้างโปรเจกต์ใหม่และนำเข้าโปรเจกต์นี้
3. กำหนดค่าสภาพแวดล้อม **Environment Variables** บน Vercel:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. กด **Deploy** และรอรับโดเมนใช้งานจริงได้ทันที

## 🔒 บันทึกความปลอดภัยเพิ่มเติม (Security Notes)

- **SQL Injection Prevention**: การดึงข้อมูลทุกจุดกระทำผ่าน Supabase JavaScript SDK (PostgREST API) ซึ่งส่งชุดคำสั่งที่ป้องกัน SQL Injection แบบ Parametrized Queries มาให้ในตัว
- **Cross-Site Scripting (XSS)**: การแสดงผลข้อความคำอวยพรและโปรไฟล์ทั้งหมดถูกเขียนทับลงใน DOM ผ่านการใช้ `textContent` และตัวแปรสตริงที่ถูก Escape ผ่านบราวเซอร์ หลีกเลี่ยงการแทรก `innerHTML` ที่มีความเสี่ยง
- **CSRF & Session Hijacking**: Supabase Auth ใช้ JWT Token จัดเก็บลงใน LocalStorage ของบราวเซอร์ผู้ใช้โดยตรง มีการหมดอายุตามระยะเวลา และส่งหัวข้อผ่าน HTTPS Header ในทุก Requests ที่ติดต่อฐานข้อมูล
